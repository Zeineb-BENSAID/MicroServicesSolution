## Hands on the Application

Clone the repository along with all submodules:

```bash
git clone --recurse-submodules https://github.com/Zeineb-BENSAID/ProductServiceSolutionFinal.git
```
This solution is designed as a product management system.
Its main goal is to provide robust CRUD (Create, Read, Update, Delete) operations and advanced querying capabilities for products, suitable for business or e-commerce platforms.

Main Components
<details> <summary>1. Domain Layer</summary>

Entities

Product: Represents the core business object managed by the application.

</details> <details> <summary>2. Data Access Layer</summary>

DbContext

ProductDbContext: Manages database connections and entity sets using Entity Framework Core.

Repositories

GenericRepository<T>: Implements generic data access logic for any entity.

ADProductService: Inherits from the generic repository and provides advanced, non-CRUD operations for products.

</details> <details> <summary>3. Application Layer</summary>

Services

ProductService: Contains business logic for products, including filtering, pagination, and mapping between entities and DTOs.

Interfaces

IProductService

IADProductService: Define contracts for service implementations, supporting both standard and advanced operations.

</details> <details> <summary>4. DTOs (Data Transfer Objects)</summary>

ProductDto: Used to transfer product data between layers, ensuring separation between internal models and external contracts.

</details> <details> <summary>5. Infrastructure Layer</summary>

Data: Handles the actual database operations and context management.

</details>


## Features

- Fully functional CRUD operations for products
- Advanced filtering and pagination for product listings
- Layered architecture ensuring separation of concerns
- Use of Entity Framework Core for database management
- DTOs for clean separation between internal models and external contracts
- Extensible repository pattern allowing easy addition of new business logic
